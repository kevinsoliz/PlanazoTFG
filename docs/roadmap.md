# Roadmap de Planazo 

Este documento define las fases de desarrollo previstas para la aplicación **Planazo**, desde la creación inicial de la landing page hasta la implementación de funcionalidades clave conectadas a una base de datos.

---

## Fase 1 — Base del Proyecto (Estado: En progreso)
**Objetivo:** Sentar los cimientos del proyecto.

### Tareas:
- Crear la estructura inicial del repositorio.
- Configurar carpetas: `public/`, `src/`, `docs/`.
- Preparar el entorno de trabajo con SCSS y JavaScript modular.
- Diseñar y construir la **Landing Page**:
  - Componentes básicos
  - Hero con CTA
  - Estilos iniciales
  - Versión responsive

---

## Fase 2 — Página de Login (Pendiente)
**Objetivo:** Crear el acceso básico de usuarios.

### Tareas:
- Maquetar la página de Login.
- Crear componentes:
  - Formularios
  - Inputs
  - Botón CTA
- Preparar la estructura para futura validación.
- Añadir estilos SCSS organizados por módulos.
- Preparar lógica JS inicial (aún sin backend).

---

## Fase 3 — UI Principal de la App (Pendiente)
**Objetivo:** Construir la interfaz donde los usuarios podrán ver y crear planes.

### Tareas:
- Diseñar la vista principal de usuario (dashboard minimalista y amigable).
- Crear componentes:
  - Barra superior
  - Lista de planes
  - Botón para crear nuevo plan
  - Componentes de usuario / avatar
- Añadir interactividad básica con JavaScript (aún sin conexión real).

---

## Fase 4 — Backend y Conexión a Base de Datos (Pendiente)
**Objetivo:** Implementar la conexión real entre frontend, backend y base de datos.

### Tareas:
- Configurar backend con **PHP**.
- Crear la primera base de datos (MySQL recomendado).
- Implementar endpoints para:
  - Registro y login de usuarios
  - Creación de planes
  - Listado de planes
  - Gestión de usuarios
- Conectar el frontend a estos endpoints mediante fetch o AJAX.
- Probar la comunicación entre frontend y backend.

---

## Fase 5 — Funcionalidades Principales (Pendiente)
**Objetivo:** Hacer la app funcional.

### Funciones mínimas:
- Crear un plan.
- Unirse a un plan.
- Ver los planes creados por amigos.
- Notificaciones simples (in-app).
- Perfil básico de usuario.

---

## Fase 6 — Optimización y Versión 1.0 (Pendiente)
**Objetivo:** Refinar y publicar.

### Tareas:
- Mejorar diseño visual.
- Revisar accesibilidad.
- Corregir bugs.
- Optimizar tiempos de carga.
- Preparar despliegue.
- Crear documentación final.

---

## Objetivo Final
Publicar la **versión 1.0 de Planazo**, una app amigable y sencilla para crear y compartir planes con gente cercana.
